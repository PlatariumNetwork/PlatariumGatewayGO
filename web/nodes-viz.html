4. Network nodes (L1/L2 - real voting)

Decentralized network: it does not matter which node receives the transaction - it goes to the shared mempool and is synchronized between all nodes. Nodes take TX from mempool; the reputation-based protocol (selection weight) chooses who performs L1 (block collect) and L2 (confirmation).

L1: block is collected only if ≥67% “yes”. L2: block is confirmed only if ≥70% “yes”. Votes flow through peers; reputation is updated from votes.

4.2 Node rating (Core formula)

Reputation = Uptime×30% + Latency×20% + VoteAccuracy×30% + Stake×20%. Selection weight = reputation×(1−load). Values 0…1.
